# 许愿示例 (Wish Examples)

以下是一些高质量许愿的示例，供参考：

## 示例 1: 功能增强许愿

**标题**: 添加 MACD 金叉死叉策略模板

**描述**: 
希望在策略库中添加基于 MACD 指标的经典交易策略模板，包括：
- MACD 金叉买入信号检测
- MACD 死叉卖出信号检测
- 可配置的 MACD 参数（快线、慢线、信号线周期）
- 集成止损止盈机制
- 支持多时间周期分析

**使用场景**: 
量化交易初学者可以快速上手经典技术分析策略，也可作为复合策略的组件使用。

**期望效果**: 
用户可以通过简单配置就能运行 MACD 策略回测，并在此基础上进行优化改进。

## 示例 2: 问题修复许愿

**标题**: 数据接口在市场闭市时返回空数据

**描述**: 
当前数据接口在股市闭市时间调用会返回空数据集，但没有明确的错误提示，容易导致策略运行异常。

**重现步骤**:
1. 在非交易时间调用 `get_price()` 函数
2. 函数返回空的 DataFrame
3. 策略代码因数据为空而异常退出

**期望行为**: 
应该返回最近一个交易日的数据，或提供清晰的状态码和错误信息。

**建议修复方案**: 
在数据接口中添加市场状态检查，并提供配置选项让用户选择处理方式。

## 示例 3: 创意想法许愿

**标题**: 集成机器学习自动特征工程功能

**描述**: 
基于历史交易数据，自动生成和筛选有效的技术指标组合，用于策略优化。

**创新点**:
- 使用遗传算法自动组合技术指标
- 基于信息增益自动筛选有效特征
- 支持自定义评估指标和约束条件

**潜在收益**: 
大幅减少策略开发时间，提高因子挖掘效率，降低过拟合风险。

**实现思路**: 
可以基于 scikit-learn 和 DEAP 库实现，提供简洁的 API 接口。

## 许愿模板快速填写指南

### 1. 选择合适的许愿类型
- 新功能 → 功能增强许愿
- 修复问题 → 问题修复许愿  
- 策略需求 → 策略相关许愿
- 创新想法 → 创意想法许愿

### 2. 撰写清晰的标题
- 使用动词开头 (如: 添加、修复、优化、支持)
- 包含关键词让他人快速理解
- 控制在 50 字符以内

### 3. 详细描述需求
- 背景和动机
- 具体功能点
- 技术要求
- 参考资料

### 4. 说明使用场景
- 什么情况下使用
- 目标用户群体
- 解决什么问题

### 5. 描述期望效果
- 实现后的具体效果
- 对用户的价值
- 可衡量的指标

记住：高质量的许愿更容易被采纳和实现！
